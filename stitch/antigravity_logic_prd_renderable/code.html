Technical Specification

PRD: Antigravity Data Orchestrator

1. Project Overview

Role: Data orchestration engine and logical middleware for Antigravity (Refrigeration Distributor, Miami, FL).

Objective: Configure data flows, cross-border validations, and event triggers to connect external logistics events with the Antigravity backend.

2. Integration Specifications

2.1 Inbound Webhook Mapping (Maritime Tracking)

  • Endpoint: /v1/webhooks/vessel-tracking
  • Transformation Logic: Convert all timestamps from UTC to US/Eastern (Miami Time).
  • Status Normalization:
    • status: 3 → IN_TRANSIT
    • status: 5 → CUSTOMS_MIAMI
  • Alert Rule: Trigger LOGISTICS_DELAY_CRITICAL if new_eta > old_eta by more than 48 hours.

2.2 Data Contracts & Validation

  • Shipment Schema: container_number must follow ISO 6346 (4 letters, 7 numbers).
  • Tax Engine: Default sales_tax_rate: 0.07 (Miami-Dade). Injected to all Order objects unless customer_tax_exempt is true.

2.3 Synchronization Triggers

  • Event: ARRIVED_PORT → Send payload to PATCH /inventory/update-status.
  • Cost Integration: On transition from CUSTOMS to ARRIVED, calculate Landed Cost = fob_price + customs_taxes_miami + inland_freight.

2.4 Exception Handling

  • Diversion Check: If destination_port is not "PortMiami (USMIA)" or "Port Everglades (USPEF)", flag as DIVERSION_WARNING and block stock updates.
